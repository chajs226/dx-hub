## 1. 프로젝트 개요

병원의 **디지털정보혁신본부**에서 그동안 수행한 AI 사업 및 연구 성과를 한눈에 보여주는  
**웹 기반 아카이브 서비스**를 구축하고자 합니다.

- 내부 보고 및 외부 홍보용으로 활용
- 각 프로젝트의 상세 화면은 별도 수정 없이 **발표 자료(Presentation)**로 바로 사용 가능해야 함
- 가독성과 시각적 임팩트를 중시한 UI/UX 설계 필요

---

## 2. 레이아웃 및 구성 요소

전체 화면은 **고정 레이아웃(Fixed Layout)** 기반의 대시보드 형태로 구성하며,  
스크롤 없이 한 화면 내에서 모든 주요 인터랙션이 이루어지도록 설계합니다.

---

### 2.1 Top Filter Bar (상단 영역)

- **기술 영역 필터 버튼**
  - 전체
  - 의료영상
  - 자연어처리
  - 예측분석
- **연도 선택 UI**
  - 드롭다운 또는 슬라이더 형태

---

### 2.2 Left Sidebar (좌측 영역)

- **세로형 타임라인 뷰**
  - 선택된 필터 조건에 따라 프로젝트 리스트를 연도별로 정렬
- **리스트 항목 구성**
  - `[연도] - [프로젝트명] - [기술 아이콘]`
- **선택 상태 표현**
  - 현재 선택된 프로젝트는 하이라이트 처리
- **AI 영역별 테마 컬러 적용**
  - 예:  
    - 의료영상 → Blue  
    - 자연어처리 → Green  
  - 적용 위치: 왼쪽 보더(Border) 또는 아이콘 컬러

---

### 2.3 Main Content Section (중앙 / 우측 영역)

- 좌측 타임라인에서 선택한 프로젝트의 **상세 정보 표시**
- **발표 모드(Presentation Mode) 레이아웃**
  - 큼직한 프로젝트 제목
  - 연구 배경 설명
  - 핵심 성과(KPI) 강조
  - 기술 스택 태그(Tag) 형태로 표시
- **미디어 중심 구성**
  - 이미지 또는 영상이 화면의 큰 비중을 차지
  - 클릭 시 **확대(Viewer) 기능** 제공

---

## 3. 기술 요구사항

- **Framework**
  - React (Vite 환경 권장)
- **Styling**
  - Tailwind CSS
  - 의료 서비스에 어울리는 **Blue / Slate 계열 톤**
  - Glassmorphism 스타일 일부 적용
- **Icons**
  - `lucide-react` 또는 `FontAwesome`
- **Animation**
  - 프로젝트 선택 시 부드러운 전환 효과
  - `Framer Motion` 사용 선호
- **Data 관리**
  - 프로젝트 데이터는 JSON으로 분리하여 관리
  - 해당 데이터를 기반으로 화면 렌더링

## 4. 요청 사항
- 위 구조를 기반으로 한 단일 파일(App.js) 또는 핵심 컴포넌트 구조 작성
- 상단 필터 클릭 시
    - 좌측 타임라인 리스트가 조건에 맞게 필터링되도록 구현
- 좌측 리스트 클릭 시
    - 중앙 메인 콘텐츠 영역의 프로젝트 상세 데이터가 변경되도록 상태 관리 구현
- 전체 디자인은 병원 디지털 혁신 본부에 어울리는 전문적이고 신뢰감 있으며 깔끔한 스타일로 적용
- 콘텐츠의 본문 가독성을 위해 '블록 기반 레이아웃'을 적용해줘. JSON 데이터에서 content_blocks라는 배열을 읽어서, 타입이 text면 본문 폰트로 보여주고, image나 video면 화면 너비에 맞게 미디어를 배치해줘. 이렇게 하면 내가 텍스트 중간에 이미지를 자유롭게 넣을 수 있을 것 같아. 예시 데이터: data/projects_data.json
